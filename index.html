02 · Problem Framework

Situation · Complication
· Question

Situation, complication, question. Frame the problem before solving it. No hypothesis without first establishing what is true, what is broken, and what we need to answer.

Situation

Cryptio is a digital asset accounting platform serving exchanges, funds, and corporate treasury clients across EU, UK, and US. Client 2 has seven connected sources including Coinbase, Gemini, Deribit, and Fireblocks. Books are live. Audit is six months away. The platform is generating Missing Volumes errors: assets recorded as disposed with no matching acquisition cost.

Complication

Missing volume errors corrupt the cost basis calculation. Every downstream output, P&L, trial balance, and tax report is wrong. The client is a regulated exchange. Their financial statements and regulatory filings depend on clean books. The root cause is still unknown. Until we know whether this is a configuration issue or a misclassification, we cannot fix it.

Key Question

What is the root cause of the Missing Volumes and what is the fastest path to fix the errors before the audit?

01
Phase 1 — Scope the Error · Day 1
Export full Missing Volumes report. Identify the pattern. Send client update.
Export asset, quantity, disposal date, proceeds from Cryptio. Identify whether errors cluster on specific assets, date ranges, or transaction types (sales vs transfers vs swaps). Cross-reference against Coinbase import logs: confirm configured start date, check for sync failures, API rate limit errors, pagination gaps.
-- Scope: Missing Volumes by asset
SELECT asset, COUNT(*) as errors,
  MIN(disposal_date) as earliest,
  SUM(proceeds) as exposed_value
FROM missing_volumes_report
GROUP BY asset ORDER BY exposed_value DESC
02
Phase 2 — Diagnose · Day 1–2
Test all three hypotheses. Document root cause in a pager doc before proposing a fix.
Test A: compare earliest disposal date vs Coinbase source start date. Test B: check each affected asset on every other connected source, Gemini, Fireblocks, OTC. Test C: review inbound Coinbase transactions, confirm how Cryptio classified them as transfer or deposit. Share findings with the client team before suggesting a fix. This shows we are on top of it.
03
Phase 3 — Resolve · Day 2–4
Execute fix based on confirmed root cause.
A: extend Coinbase start date, re-sync, verify transaction count difference in logs, re-run cost basis. B: import originating source; create manual cost entry for OTC with full documentation (date, quantity, unit cost, counterparty, receipt). C: re-tag as Transfer In, re-run matching, verify Transfer Out at origin. I own step C. Client reviews and signs off.
04
Phase 4 — Validate, Validate Again and Document · Day 4–5
Zero Missing Volumes in logs. Written audit trail delivered.
Re-run cost basis report. Confirm Missing Volumes count = 0 for affected assets. Share a written resolution TLDR: root cause confirmed, steps taken, what the client should monitor. This also serves as the audit trail. Prevention: implement a Source Completeness Checklist alert as Step 1 of every monthly close. Long term, learnings should be automated and captured in an onboarding playbook.

05 · Strategic Recommendations

Three Pillars.
Each One a Call to Action.

Not topic headers. Calls to action. Every sentence either advances the fix or prevents the next fire.

Pillar 1
Fix the data before it enters the audit record
  • The 6-month audit window creates a hard deadline. Every close cycle running on corrupted data makes remediations harder and the audit conversation harder.
  • Sequence: scope, diagnose, resolve, validate. A fast guess that is wrong costs more than a methodical two-day investigation.
Pillar 2
Prevent recurrence through a structured close process
  • Most Missing Volumes cases come from configuration gaps during the integration phase. A Source Completeness Checklist before every cost basis run catches them early. Low effort, high impact.
  • The close runbook is a living document. Update it every time a new source or asset type is added.
Pillar 3
Use Client 1 advisory to deepen the strategic relationship
  • FIFO to HIFO is a serious accounting decision with tax consequences that are hard to reverse. Not a support ticket. Prepare a structured options brief with rationale.
  • A $20K ARR advisory handled well can lead to $50K ARR. Deliver in a dedicated call with the Account Manager present.
⚠ Risk: Escalate Immediately
If Hypothesis B is confirmed and the client has no OTC acquisition documentation, escalate to the right channels immediately for visibility. The client must make a formal accounting policy decision on a deemed cost basis before the next close. This carries direct tax consequences and must be documented before entering their audit record.
